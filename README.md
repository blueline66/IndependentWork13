## Самостійна робота №13
## Тема: Використання Polly 
## Опис проєкту

Проєкт IndependentWork13 демонструє застосування бібліотеки Polly у .NET для підвищення відмовостійкості застосунків. Реалізовано три сценарії з використанням політик Retry, Circuit Breaker і Timeout.
Усі сценарії імітують реальні проблеми через штучні помилки та затримки.
## 1: Виклик зовнішнього API 

Проблема: зовнішній API може тимчасово бути недоступним.
Політика: Retry з експоненційною затримкою (2 → 4 → 8 секунд).
Очікувана поведінка: перші дві спроби падають, третя — успішна; у консолі відображаються повтори.

## 2: Підключення до бази даних 

Проблема: під час тимчасової недоступності БД багато повторних спроб збільшують навантаження.
Політика: CircuitBreaker — після двох помилок переходить у стан Open на 5 секунд.
Очікувана поведінка: після двох невдач виклики блокуються, потім переходить у Half-Open та виконує тестову спробу.

## 3: Довготривала операція 

Проблема: операція може зависнути або виконуватись надто довго.
Політика: Timeout (3 секунди), песимістична стратегія.
Очікувана поведінка: операція триває довше 3 секунд, тому Polly перериває її із винятком TimeoutRejectedException.

 ## Висновки

Polly значно підвищує стабільність і відмовостійкість застосунків.
Використання політик повторів, автоматичного розриву ланцюга та тайм-аутів дозволяє коректно обробляти помилки, уникати перевантаження сервісів та запобігати зависанню операцій.

# IndependentWork12: PLINQ — дослідження продуктивності та безпеки

## Опис проєкту
Цей проєкт демонструє використання PLINQ (Parallel LINQ) для паралельної обробки колекцій у порівнянні зі звичайним LINQ.  
Мета — оцінити переваги та недоліки PLINQ щодо продуктивності та безпеки, а також продемонструвати потенційні проблеми потокобезпеки.

## Основні завдання
1. Створення великої колекції даних (List<int>) з 1M, 5M та 10M елементів.  
2. Виконання обчислювально інтенсивної операції над кожним елементом:
   - Перевірка на просте число (IsPrime)  
   - Обчислення квадратного кореня (Math.Sqrt)  
3. Порівняння продуктивності LINQ та PLINQ.  
4. Демонстрація потенційних побічних ефектів при паралельній обробці та їх виправлення за допомогою lock.  

## Технології
- C# (.NET 6 або вище)
- Visual Studio або VS Code
- PLINQ (System.Linq.Parallel)
- Stopwatch (System.Diagnostics.Stopwatch)
